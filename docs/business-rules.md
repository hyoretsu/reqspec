# Business Rules

This file is the source of truth for ReqSpec product and domain rules.

Use this file for rules that describe how the product should behave: request execution, collection structure, environment variable resolution, imports, scripts, history, storage, and runner behavior. Keep engineering workflow, coding style, and agent operating rules in `AGENTS.md`.

For implementation context, start with the focused feature docs under `docs/features/` and keep any cross-feature product rule mirrored or summarized here when it becomes normative.
